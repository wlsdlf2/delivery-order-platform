# 프로젝트 개요 (Project Overview)

## 프로젝트 목적

광화문 근처에서 운영될 음식점들의 **배달 주문 관리, 결제, 주문 내역 관리** 기능을 제공하는 플랫폼입니다.

---

## 서비스 흐름

```
PENDING(주문요청, CUSTOMER)
  → ACCEPTED(주문수락, OWNER)
    → COOKING(조리완료, OWNER)
      → DELIVERING(배송수령, OWNER)
        → DELIVERED(배송완료, OWNER)
          → COMPLETED(주문완료, OWNER)
```

---

## 사용자 역할(Role) 및 권한

| 역할 | 설명 | 주요 권한 |
|------|------|-----------|
| `CUSTOMER` | 일반 고객 | 자신의 주문 내역만 조회 가능 |
| `OWNER` | 가게 주인 | 자신의 가게 주문 내역 조회, 가게 정보·메뉴 수정, 주문 처리 가능 |
| `MANAGER` | 서비스 담당자 | 모든 가게 및 주문에 대한 전체 권한 보유 |
| `MASTER` | 최종 관리자 | MANAGER 권한 포함 + MANAGER 계정 생성·조회·수정·삭제 가능 |

> **Note:** MASTER는 선택 구현 항목입니다.

---

## 핵심 기능 요약

- 음식점 및 메뉴 등록/관리
- **AI 상품 소개 추천**: 상품(메뉴) 등록 시 AI를 통해 소개 문구 자동 추천
- 주문 생성 및 관리
- **주문 취소 제한**: 주문 생성 후 5분 이내에만 취소 가능
- 배달 상태 추적
- JWT 기반 인증 및 역할별 접근 권한 관리
